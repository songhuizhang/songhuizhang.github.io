《JavaScript高级程序设计》---引用类型

引用类型的值是引用类型的一个实例。对象是某个引用类型的实例。新对象是使用new操作符后跟一个构造函数来创建的。构造函数本身就是一个函数，只不过它的目的是出于创建一个新对象。

使用构造函数Object为新对象定义了默认的属性和方法。

Object 类型

Object 是 JavaScript 中使用最多的一个类型，创建object实例的方式有两种：

访问对象属性

一般来说，访问对象属性都用点语法，但是在一些情况需要使用方括号法来访问对象的属性。方括号法的有点是可以用过变量来访问属性。

Array 类型

Array 类型是数据的有序列表，他的每一项都可以保存任意类型的数据，创建Array 也有两种：

检测数组

  1. value instanceof Array ==>boolean
  2. Array.isArray( value ) ==> boolean
  3. Object.prototype.toString.call( value ).slice(8,-1) ==> "Array"

转换方法

toString() , valueOf() 是将数组转换为以逗号分割的字符串，如果想使用其它的分割符可以使用value.join('||');

数组操作方法：

  • push() pop() 是对数组最后一个项的操作；
  • shift() 取出数组的第一个，原数组数量减一；
  • unshift() 想数组前推入任意长度的项，返回新长度的数组；
  • reverse() 和 sort() 第一个是反转排序，第二个是按升序排列；
  • concat() 拼接数组合为一个，不会改变原数组
  • slice() 截取数组一部分返回，不会改变原数组
  • splice() 它有多种用法，主要是用来向数组中部插入项。
    • 删除：splice(0,2) 删除数组前两项
    • 插入 splice(2,0,”red”,”green”)； 从位置2开始插入字符串red greeb
    • 替换 splice(2,1,”red”,”green”)； 从位置2开始删除2量，再插入2项字符串red greeb
  • every() 对数组的每一项运行给定的函数，如果该函数对每一项都返回true则返回true
  • filter() 对数组的每一项运行给定的函数，该函数会返回true的项组成的数组
  • forEach() 对数组的每一项运行给定的函数，无返回值
  • map() 对数组的每一项运行给定的函数，返回每次函数调用后的结果组成的数组
  • some() 对数组的每一项运行给定的函数，如果该函数对任意一项返回true则返回true

《JavaScript高级程序设计》---变量和作用域

在JavaScript中变量是松散类型，只是在特定时间保存特定的值的一个名字而已。但是由变量引出的一些问题就很有意思也很重要了，如：作用域链，变量提升，变量的拷贝。

前言

在《JavaScript高级程序设计》（第三版）中，变量类型分为基本类型和引用类型两种，ES6的新增类型以后再谈。

  • 基本数据类型：undefined、null、boolean、number、string是按值访问的。
  • 引用类型是按照引用地址访问的，此类型可以动态的添加属性

变量的复制

基本类型

此类的复制，就只把改变了的值复制到新分配的变量位置上

上面的代码在栈内存中实际的过程是

复制后
num1 5
num2 5

引用类型

此类的复制，就是栈内存中值的副本实际上是堆内存一个对象的指针，复制后新老变量实际将引用同一个对象。因此，改变其中一个变量，另一个也会改变；

下图表示了变量的对应关系

传递参数

在JavaScript中所有函数的参数都是按值传递的，也就是说把值从一个变量复制到另一个变量。

基本类型的变量没得说，关键就是引用变量的传递。

引用变量的传递会把这个值对应内存中的地址复制给一个局部变量，因此在内部修改该局部变量，外部的对应变量也会修改。对应代码如下：

检验变量类型

使用typeof操作符可以检验变量是字符串、数值、布尔值、还是undefined的最佳工具；

最优检验法：

执行环境

执行环境定义了变量或函数有权访问其它的数据，决定了他们各自的行为；

这里有几种穿件的执行环境：

  • web浏览器中全局执行环境指的是window对象；
  • 函数都有自己的执行环境，函数的形参的执行环境也是函数执行环境；

作用域链

当代码在一个环境中执行的时候，会创建变量对象的一个作用域链，它的作用是保证对执行环境有权访问的所有变量和函数的有序访问；

作用域链是由内向外一直到全局执行环境的一个访问关系；在任何环境下都不能向下搜索作用域从而进入另一个执行环境；

每次进入一个新的执行环境，都会创建一个用于搜索变量和函数的作用域链

块级作用域

这一部分在es6中，有关键字const和let创建的变量都拥有块级作用域。这里不细致展开；

在JavaScript中一定要遵循先申明后引用的规则。

关于需求的理解和解决方案的选择漫谈

需求的理解

在日常的开发中，我习惯的是先对需求进行分析，对于需求的实现程度和截止时间有一个认识，而不是拿到需求草草看一下就直接上手开发。

对于需求分析，我有一下几点经验：

  1. 需求合理性和实现性的大致评估
  2. 单个需求在于整个项目中是否有其它关联
  3. 需求前后是否进行闭环验证
  4. 开发时间是否合理

需求合理性和实现性

此需求对应的功能上线时间，是否定稿，后期变动范围。

在一般的公司中，该需求是十分普遍的事情，对于这一点我想大多数的技术都十分的排斥。如果是我们是甲方，那么项目经理和产品经理会对所有的需求的上线计划进行排期，排重。但是如果我们公司处于乙方公司，项目经理和产品经理面对客户需求是出于被动的地位，这是开发更是处于被动的被动地位，那么我们开发就是乙方的乙方了。

我在这里就所以下作为乙方的乙方我们应该如何去做。首先我们和产品经理、项目经理都是同事的关系，而且初衷都是希望将产品做的符合甲方的要求，所以我们不是敌对的关系。我们此时能做的就是尽可能的帮助确认需求的合理性和提供需求的解决方案。

判断需求的关联性

这一点是不分公司性质的，需求的关联性在任何情况下作为前端都因该帮忙去确认前后关联性和影响范围。

如何做呢？了解业务，没做不了解业务的前端不是好的前端，不管我们是刚入职还是老员工，我们都应该尽可能的去了解我们涉及到的业务。业务流程熟悉的前提下，才能尽可能的判断需求的关联性。

了解之后我们在以后的开发中就可以避免以后需求覆盖的问题。

需求前后是否进行闭环验证

需求的闭环验证，数据进来后的处理，验证，下一步，逻辑等等。如果我们开发的不是一个闭环需求，那么相信我后期的麻烦会绵绵不断的。

可能有很多人会说：“我是开发我就只管开发的事情。“

话虽如此，但是我的个人经验来说，在产品经理普遍业余的大环境下，不了解需求的开发不是一个好的开发。

开发时间是否合理

对于开发时间紧的情况下，作为拒绝加班党的我会如何做呢？

分析需求的最低要求，已经最快实现方案。大多数人可能会说：”不一次做好的东西后期改动起来会更痛苦的。“没错，但是我就是不想加班怎么办呢？一般来说开发时间的不合理是由很多因素造成的，所以这类需求本来就充满了不确定性。大概率的情况下，改动是必然的。所以你懂的，解放自己的个人时间。

以上是我的个人经验，欢迎探讨交流，拒接直接喷！！

秋，腰，自律

伤痛

最近半个月的时间腰尽然连续扭伤两次，让年仅25岁的我想到一首歌《往事只能回味》。

歌词写的好时光已逝永不回,往事只能回味虽然这首歌是说两小无猜，青梅竹马的往事恋情。

但是此前此景此腰让我有着另一番感受:

大学的时候正值青春年华，那会儿的我是不知疲惫为何物。（身体和心理）

现在毕业两年时间，体重涨了20斤，本该充满活力的身体也应为久坐和姿势的问题变得容易受伤。

有句名言：你才25岁，你可以成为任何你想成为的人。

我一直对此话深信不移，人的一生有着无限的可能性，只要努力就一定可以比现在过的更好的。

但是，还有一句名言身体是革命的本钱

现在的我体重微胖、腰肌劳损、作息不规律，没有任何革命的本钱。现在都在说90后已经步入养身大军，这月的两次受伤让我真心觉得原来不在意的事情，尽然在慢慢侵蚀着我的身体。

自律

人都说不要常立志要立长志。

我喜欢追求自由，但是现在看似自由的我正在被身体的伤痛，口腹的欲望所缠绕。有一期锵锵三人行中，窦文涛说过这么一句话：“只有自律的人，才是真正自由的人。”。

自律是一种生活的态度，看似给生活加了许多的条条框框来约束自己。但是，让你习惯后你会发现这些条条框框不是对自我的约束，而是一种促使自己实现自我目标的动力。

我的目标是人生的自由，这需要有一个健康的身体，有着足够的金钱；在拥有这些的前提下，才能去尝试着追寻自我的自由。

结语：希望明天的自己会满意现在自己走的每一步，现在的自己更要努力不让明天的自己失望！！
