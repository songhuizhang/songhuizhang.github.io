《JavaScript高级程序设计》---面向对象的程序设计(1)

发布 : 2018-11-25 分类 : JavaScript 浏览 :

面向对象语言都有一个标志，那就是类。在JavaScript中没有类的概念，因此它实现面向对象的和传统的面向对象语言有所不同。

术语

Namespace 命名空间

允许开发人员在一个独特, 应用相关的名字的名称下捆绑所有功能的容器。

在 JavaScript中是一个包含方法、属性和对象的对象。

值的注意的是：JavaScript的普通对象和命名空间在语言上是没有区别的。

创造命名空间的想法很简单：一个全局对象被创建，所有的变量，方法和功能成为该对象的属性。使用命名空间也最大程度地减少应用程序的名称冲突的可能性。

Class 类

定义对象的特征。它是对象的属性和方法的模板定义.

JavaScript是一种基于原型的语言，它没类的声明语句。但是，可用方法作类，定义一个类跟定义一个函数一样简单。

Object 对象

类的一个实例。

Property 属性

对象的特征，比如颜色。

属性就是 类中包含的变量;每一个对象实例有若干个属性. 为了正确的继承，属性应该被定义在类的原型属性 (函数)中。

Method 方法

对象的能力，比如行走。

方法与属性很相似， 不同的是：一个是函数，另一个可以被定义为函数。 调用方法很像存取一个属性, 不同的是add () 在方法名后面很可能带着参数. 为定义一个方法, 需要将一个函数赋值给类的 prototype 属性; 这个赋值给函数的名称就是用来给对象在外部调用它使用的。

方法是一个普通函数，所以它可以在context之外调用。

上述例子也说明了this是代表当前对象
当从变量 helloFunction()中调用的时候， this就被设置成了全局对象 (在浏览器中即window)
可以使用apply或call显性的是这this的值。
helloFunction.call(person1); // alerts "你好，我是张三"

Constructor 构造函数

对象初始化的瞬间, 被调用的方法. 通常它的名字与包含它的类一致.

在JavaScript,中函数就可以作为构造器使用,因此不需要特别地定义一个构造器方法. 每个声明的函数都可以在实例化后被调用执行

构造器常用于给对象的属性赋值或者为调用函数做准备。

Person类实例化时构造器调用一个 alert函数

Inheritance 继承

一个类可以继承另一个类的特征。

在Javascript中，继承通过赋予子类一个父类的实例并专门化子类来实现。

下面的例子中, 我们定义了 Student类作为 Person类的子类. 之后我们重定义了sayHello() 方法并添加了 sayGoodBye() 方法.

Encapsulation 封装

一种把数据和相关的方法绑定在一起使用的方法.

Student类虽然不需要知道Person类的walk()方法是如何实现的，但是仍然可以使用这个方法；Student类不需要明确地定义这个方法，除非我们想改变它。 这就叫做封装，对于所有继承自父类的方法，只需要在子类中定义那些你想改变的即可。

Abstraction 抽象

结合复杂的继承，方法，属性的对象能够模拟现实的模型。

Polymorphism 多态

多意为‘许多’，态意为‘形态’。不同类可以定义相同的方法或属性。

就像所有定义在原型属性内部的方法和属性一样，不同的类可以定义具有相同名称的方法;方法是作用于所在的类中。并且这仅在两个类不是父子关系时成立（继承链中，一个类不是继承自其他类）。

理解对象属性

创建自定义对象

方法一：通过Object创建一个实例，再为它添加属性和方法；

方法二：通过字面的方法创建对象—-手动的写出对象的内容来创建一个对象.

当你想要传输一些有结构和关联的资料时常见的方式是使用字面量来创建一个对象
关键字”this”指向了当前代码运行时的对象

属性类型

ECMAScript中有两种属性：数据属性和访问器属性

数据属性

包含一个数据值的位置，这个位置可以读取写入，它有4个描述行为的特征

  • [[Configurable]]:表示是否具有可以删除和可修改的特性，默认值：true
  • [[Enumerable]]:表示是否具有可以通过for-in返回的属性，默认值：true
  • [[writable]]:表示包含可以修改的属性的值，默认值：true
  • [[value]]:表示包含这个属性的数据值，默认值：undefined

修改默认属性特性，需要通过Object.defineProperty()方法，但是当[[Configurable]]为false后，就不能在修改属性特征了；

访问器属性

不包含数据值，只包含了一对getter和setter函数，再读取访问器属性时调用了getter，写入时调用了setter，访问器属性有如下四个特征

  • [[Configurable]]:表示是否具有可以删除和可修改的特性，默认值：true
  • [[Enumerable]]:表示是否具有可以通过for-in返回的属性，默认值：true
  • [[Get]]:读取属性时调用的函数，默认值：undefined
  • [[Set]]:写入属性时调用的函数，默认值：undefined

访问器属性不能直接定义，需要通过Object.defineProperty()来定义

定义多个属性

可以使用Object。defineProperties()一次定义多个属性。

在详细展开之前，先对面向对象的三个特征要一个了解封装、继承、多态

封装

封装用于阐述对象所包含的内容，通常由两部分组成

  • 相关的数据（用于储存属性）
  • 基于这些数据所能做的事（所能调用的方法）

继承

通过继承这种方式，我们可以非常优雅的实现对现有代码的复用。在JavaScript中继承就是用过对对象的继承来实现的。

多态

不同对象通过相同的方法调用来实现各自行为的能力叫多态

创建对象

虽然开始提到的Object构造器和字面量的方式都可以用来创建对象，但是使用同一个接口创建很多对象，会产生大量的重复代码。
为了使代码更有效率就引出了设计模式，这里只说JavaScript高程中和创建对象有关的设计模式，其他的在以后会单独细谈。

  • 工厂模式
  • 构造函数模式
  • 原型模式
  • 组合使用构造函数和原型模式
  • 动态原型模式
  • 寄生构造函数模式
  • 稳妥构造函数模式

工厂模式
